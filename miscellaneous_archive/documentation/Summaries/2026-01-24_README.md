# ML-Signals: MetalMind SMCForge

**Institutional Trading Intelligence Platform** - Real-time trading signals powered by XGBoost and Smart Money Concepts (SMC).

## 🚀 Quick Start

### Start the Application
```powershell
# From the ml-signals directory:
.\start_all.ps1

# Or start backend directly:
.\start_backend.ps1
```

### Access the Application
- **Main Application**: http://localhost:5000
- **API Endpoints**: http://localhost:5000/api/*

## 🔧 Architecture

### Backend (Flask - Port 5000)
- **API**: RESTful endpoints for predictions, backtesting, and user management
- **Authentication**: JWT tokens with refresh, Email OTP, 2FA support
- **Database**: SQLite (SQLAlchemy ORM) - easily switchable to PostgreSQL
- **Security**: Rate limiting, CORS, path traversal protection
- **Static Files**: Serves the new UI directly from Flask

### Frontend (Vanilla JS)
- **No Build Required**: Pure HTML/CSS/JavaScript
- **Modular**: OOP-based components (DashboardPage, BacktestPage, etc.)
- **Real-time**: Live predictions with auto-refresh
- **Charts**: Plotly.js for candlestick charts
- **Styling**: Tailwind CSS

## 📊 Key Features

1. **Live Predictions**: Real-time trading signals (BUY/SELL/NEUTRAL)
2. **Backtesting**: Test strategies on historical data
3. **SHAP Analysis**: Model explainability with feature importance
4. **Multi-Asset**: Support for Gold (XAU/USD) and Silver (XAG/USD)
5. **User Management**: Registration, login, profile, watchlist

## 🔒 Security Features

- JWT authentication with 15-minute access tokens
- 7-day refresh tokens (httpOnly cookies)
- Password hashing with bcrypt (12 rounds)
- Rate limiting on all endpoints
- Path traversal protection for static files
- Input validation on all user inputs

## 🛠 API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/health` | GET | Health check |
| `/api/auth/register` | POST | User registration |
| `/api/auth/login` | POST | User login |
| `/api/predictions/latest` | GET | Get latest prediction |
| `/api/backtest/results` | GET | Get backtest results |
| `/api/backtest/run` | POST | Run new backtest |
| `/api/shap/feature-importance` | GET | SHAP feature importance |
| `/api/watchlist` | GET/POST | Manage watchlist |
| `/api/profile` | GET/PUT | User profile |

## 📝 Development Notes

### Code Principles Applied
- **OOP**: All components are class-based
- **SRP (Single Responsibility)**: Services handle specific domains
- **DRY**: Centralized validation via services
- **Security**: Input validation, rate limiting, secure defaults
